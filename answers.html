Project Viva Answers
General Project Questions
1. What is the main objective of your Home Services project?
The main objective is to provide a user-friendly online platform where users can easily book trusted home and personal care services such as Electrician, Plumber, Carpenter, Painter, Cleaning, and Salon at Home. The platform streamlines the process of finding, booking, and managing home services.
2. Which technologies and frameworks did you use in this project?
The project is built using Django (Python) for the backend, HTML and Django templates for the frontend, CSS (with Bootstrap) for styling, and SQLite as the default database. Additional libraries include django-widget-tweaks for form rendering and Font Awesome for icons.
3. How is your project structured (explain the folder structure)?
The project follows Django’s standard structure:
ServiceNest/ (project root)
├── home/ (main app: models.py, views.py, forms.py, admin.py, urls.py, templates/home/)
├── static/ (CSS, images)
├── db.sqlite3 (database)
├── manage.py (Django management script)
└── ServiceNest/ (settings.py, urls.py)
4. What are the main features offered by your website?
  • User registration and login
  • Service listings for various home services
  • Booking system with unique booking numbers
  • Admin panel for managing services, bookings, and users
  • Responsive and attractive UI
  • Contact form and secure payment note
5. How does your project handle different types of services (like Electrician, Plumber, Salon, etc.)?
Each service type is represented by a model in the database. Services are managed via the admin panel and displayed dynamically on the website. Users can select the desired service when booking.
Django & Backend
6. What is a Django model? How did you use models in your project?
A Django model is a Python class that defines the structure of database tables. In this project, models are used to represent services, bookings, and other entities, allowing Django to manage data storage and retrieval.
7. Explain how you created and used the Booking model.
The Booking model includes fields for user details, service type, description, and a unique booking number. When a user submits the booking form, a Booking object is created and saved to the database.
8. How do you generate and store a unique booking number for each booking?
A custom booking number is generated in the Booking model’s save() method, combining a prefix, date, and a unique code (using uuid4). This ensures each booking can be referenced uniquely.
9. How do you handle user registration and authentication?
User registration and authentication use Django’s built-in User model and authentication system. Custom forms and views are created for registration and login, and Django’s authentication middleware manages user sessions.
10. What is the purpose of Django’s admin panel in your project?
The admin panel allows administrators to manage users, services, and bookings through a secure web interface without writing SQL queries.
11. How do you register models with the Django admin?
Models are registered in admin.py using the @admin.register decorator or admin.site.register(), enabling them to be managed via the admin panel.
12. How do you handle form validation in your project?
Form validation is handled using Django forms and ModelForms, which automatically validate required fields and data types. Custom validation logic is added in the form’s clean() method if needed.
13. How do you connect your Django project to a database?
Database settings are configured in settings.py. By default, SQLite is used, but it can be changed to PostgreSQL, MySQL, etc. Django ORM handles all database interactions.
14. What is the use of migrations in Django?
Migrations are used to apply changes to the database schema (like creating or altering tables) based on changes in models.py. They are created with makemigrations and applied with migrate.
15. How do you display dynamic data (like services or bookings) in your templates?
Views fetch data from the database and pass it to templates via context. Templates use Django’s template language to loop over and display this data dynamically.
Frontend & Templates
16. How do you use Django templates to render dynamic content?
Django templates use placeholders ({{ }}) and template tags ({% %}) to insert dynamic data and logic, such as loops and conditionals, into HTML pages.
17. How do you include static files (CSS, images) in your templates?
Static files are placed in the static/ directory. The {% load static %} tag is used in templates, and files are referenced with {% static 'path/to/file' %}.
18. How do you ensure your website is responsive and user-friendly?
Bootstrap is used for responsive design, and custom CSS ensures the site looks good on all devices. Forms and buttons are styled for usability.
19. How do you display the logged-in user’s name and handle login/logout in the navbar?
The base template checks if user.is_authenticated. If true, it displays the username and a logout button; otherwise, it shows login and register buttons.
20. How do you show success messages after a user books a service?
After a successful booking, the view passes a success flag and the booking number to the template, which displays a styled success message.
Forms & User Experience
21. How do you create and use Django forms for user input?
Django forms and ModelForms are used to define form fields and validation. They are rendered in templates and process POST data in views.
22. How do you handle CSRF protection in your forms?
Django provides CSRF protection by default. The {% csrf_token %} tag is included in all forms to prevent cross-site request forgery.
23. How do you display form errors to the user?
Form errors are displayed in the template using {{ form.errors }} or by rendering each field’s errors next to the input.
24. How do you ensure only valid data is saved to the database?
Django forms validate data before saving. If the form is not valid, errors are shown and data is not saved.
Advanced/Deployment
25. How would you deploy this project to a production server?
Deployment involves setting DEBUG=False, configuring allowed hosts, using a production-ready database, collecting static files, and deploying with a WSGI server like Gunicorn or uWSGI behind Nginx or Apache.
26. How would you add email notifications for bookings?
By configuring Django’s email backend in settings.py and sending emails in the booking view after a successful booking using send_mail().
27. How would you allow users to reset their password?
By using Django’s built-in password reset views and templates, which handle sending reset emails and updating passwords securely.
28. How would you add search or filter functionality for services?
By adding search forms and using Django ORM queries with filter() in views to display filtered results in templates.
29. How would you secure sensitive user data?
By using Django’s authentication system, HTTPS, secure password storage (hashing), and proper permissions in views and admin.
30. How would you scale your project if the number of users increases?
By switching to a more robust database (like PostgreSQL), using caching, load balancing, optimizing queries, and deploying on scalable cloud infrastructure.
